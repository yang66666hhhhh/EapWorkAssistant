# 全局 DataGrid 舒展阅读升级设计

**目标**

将项目中的主表格体验统一升级为“可读性优先”的桌面端表格：默认列宽更舒展、长文本单行截断并支持悬停查看完整内容、宽度不足时自然出现底部横向滚动条，同时保留用户手动拖拽列宽的能力。

## 背景与问题

当前项目中的 `DataGrid` 已统一使用 `ModernGrid`、`ModernGridColumnHeader`、`ModernGridCell`、`ModernGridRow` 样式，视觉基础较统一，但列宽策略仍偏保守，主要问题如下：

1. 多个主表存在较多固定窄列，导致表头与内容整体偏挤。
2. `Helpers/DataGridSmartColumns.cs` 当前仅对 `*` 列做“空内容收缩”，无法主动提供舒展的默认阅读宽度。
3. 虽然主表已启用 `HorizontalScrollBarVisibility="Auto"`，但列宽分配仍倾向于压缩内容，而不是把横向滚动视为正常体验。
4. 长文本列在首屏默认宽度不足，用户需要频繁悬停或拖拽才能读到有效信息。

## 设计目标

### 1. 阅读优先

- 默认以“列更舒展、首眼更易读”为目标。
- 不追求在中等窗口中强行容纳全部列。
- 当宽度不足时，优先保留合理列宽并显示底部横向滚动条。

### 2. 交互一致

- 所有主表统一支持用户拖拽列头手动调整宽度。
- 文本列继续采用单行截断，不引入多行展开导致的表格跳高。
- 长文本完整内容通过现有悬停预览机制呈现。

### 3. 最小改动

- 不改业务逻辑，不调整 ViewModel。
- 主要通过全局样式、列宽辅助类、主表列定义三层完成。
- 仅校准主表相关列宽，不引入额外复杂表格系统。

## 方案边界

### 本次包含

- 升级 `ModernGrid` 全局样式，使其更适合舒展阅读。
- 升级 `SmartColumns` 的默认列宽策略。
- 校准以下页面的主表列宽定义：
  - `Views/WorkRecordView.xaml`
  - `Views/IssueView.xaml`
  - `Views/DashboardView.xaml`

### 本次不包含

- 不引入冻结列、列分组、列显示/隐藏配置。
- 不修改业务数据结构。
- 不新增新的表格控件或第三方库。
- 不将长文本改为多行展示。

## 设计方案

### 一、ModernGrid 全局体验升级

在 `Resources/Styles.xaml` 中统一升级 `ModernGrid` 相关样式，使表格整体阅读感更好。

调整方向：

- 允许用户拖拽调整列宽。
- 强化表头与单元格的左右留白。
- 让横向滚动条更自然地服务于宽表，而不是作为兜底异常场景。
- 保持当前卡片化风格与项目既有视觉语言一致。

预期变化：

- 表头信息分区更清晰。
- 单元格不会因为内边距过紧而显得拥挤。
- 多列场景下，用户对“可以横向滑动”和“可以手动调列宽”的感知更明确。

### 二、SmartColumns 升级为舒展型默认列宽策略

当前 `SmartColumns` 的重点是“空内容收缩”，本次改为“舒展阅读优先”的默认列宽辅助。

核心原则：

1. 列宽不足时不主动过度压缩长文本列。
2. 长文本列提供更大的默认宽度。
3. 短字段列保持克制且稳定。
4. 用户手动拖拽后的列宽不应被频繁重置。

列宽分类策略如下：

- 紧凑列：
  - 示例：星标、状态、优先级、工时
  - 特点：宽度稳定、小而清晰

- 中等信息列：
  - 示例：任务、标题、类型、关键词、日期
  - 特点：默认比当前更宽，但不与长文本列争夺过多空间

- 长文本列：
  - 示例：内容、工作成果、问题、描述、根本原因、解决方案
  - 特点：默认给出舒适宽度，维持单行截断 + 悬停预览

- 操作列：
  - 示例：操作按钮列
  - 特点：固定宽度，避免按钮折行或拥挤

实现上优先采用“列头名称 / 绑定字段名”识别列类型，为现有页面提供稳定默认值；未命中的列则保持现有列定义，不做激进干预。

### 三、主表列定义校准

为了让全局升级立即产生明确效果，会同步调整三个主表的列定义。

#### WorkRecordView

重点：

- `任务`、`内容`、`工作成果`、`问题` 整体变宽。
- `工时`、星标、`进度`、`操作` 保持稳定宽度。
- `内容` 仍由 `DataGridCopyHelper` 负责预览，不额外增加 ToolTip。

#### IssueView

重点：

- `标题`、`描述`、`根本原因`、`解决方案` 提供更舒适的首屏宽度。
- `状态`、`优先级` 保持紧凑但不拥挤。
- `关键词` 从“可读困难”调整到“至少一眼可扫”。

#### DashboardView

重点：

- 作为摘要型表格，整体比工作记录页略克制，但仍保持舒展阅读。
- `内容`、`工作成果`、`问题` 仍需比当前更宽。
- 不因仪表盘区域宽度受限而把长文本压得难以阅读，必要时通过横向滚动解决。

## 错误处理与兼容性考虑

- 保持现有 `DataGridCopyHelper` 行为，不破坏“内容”列悬停预览。
- 不在 DataGrid 内容列上额外增加重复 ToolTip。
- 不新增 WPF 原生替代控件，仍沿用现有表格体系。
- `SmartColumns` 的调整仅针对默认布局，不介入业务数据，也不修改仓储或 ViewModel。

## 验证标准

### 修复前

- 主表列整体偏挤，尤其是长文本列可读性不足。
- 多字段表首屏阅读困难，横向滚动虽存在但体验不自然。

### 修复后

- 主表默认列宽明显更舒展。
- 当窗口宽度不足时，底部横向滚动条自然出现。
- 用户可拖拽列头手动调整宽度。
- 长文本列仍保持单行截断，并可通过悬停完整查看。

### 边界场景

- 缩小窗口宽度后，表格不应异常压缩到难以阅读。
- 数据为空时，空状态遮罩仍正常显示。
- `DashboardView`、`WorkRecordView`、`IssueView` 三处主表表现应保持一致风格。

## 受影响文件

- `Resources/Styles.xaml`
- `Helpers/DataGridSmartColumns.cs`
- `Views/WorkRecordView.xaml`
- `Views/IssueView.xaml`
- `Views/DashboardView.xaml`
