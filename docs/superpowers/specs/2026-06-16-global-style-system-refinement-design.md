# 全局样式系统精修设计

**目标**

在不改变页面结构和核心交互的前提下，保留项目现有的靛蓝、理性、专业气质，对全局设计系统进行一次高质量精修。优化范围覆盖颜色层级、阴影层级、圆角体系、基础控件尺寸节奏，以及卡片、表单、按钮、表格四大高频系统，使整体观感更高级、更统一、更稳定。

## 背景与问题

项目当前已经具备一套较完整的共享样式体系，核心样式集中在 `Resources/Styles.xaml`，页面也基本遵循统一组件和资源规范。这说明项目并不缺“样式”，真正需要提升的是“样式系统质量”。

当前主要问题体现在以下几个方面：

1. 视觉层级还不够稳定，页面中的主信息区、操作区、辅助信息区在某些场景下视觉权重接近。
2. 阴影、边框、表面色之间的分工还可以更清晰，部分区域仍偏“控件叠放”，空间感不足。
3. 高使用频率控件（按钮、输入框、下拉框、表格）虽然风格统一，但精致度和节奏感还可以继续提升。
4. 卡片、浮层、侧滑面板、表格在不同页面上的产品感已有基础，但仍存在轻微割裂，缺少“同一套系统”带来的稳定高级感。
5. 页面已具备消费级产品化基础，但在“高级感”和“成熟产品一致性”上仍有可见提升空间。

## 设计目标

### 1. 保留既有产品气质

- 不切换到新的品牌色逻辑，不改成奶油白、柔粉或完全不同的主题方向。
- 保留现有偏靛蓝、理性、专业的产品气质。
- 新设计应体现“精修升级”，而不是“换皮重做”。

### 2. 统一全局视觉基线

- 建立更稳定的颜色层级、阴影层级、圆角层级和尺寸节奏。
- 让页面中的卡片、面板、表单、表格属于同一套视觉语言。
- 控件的 hover、focus、selected 状态要更加细腻统一。

### 3. 提升高频交互区域的精致度

- 优先提升用户最高频接触的共享样式：
  - 表格
  - 按钮
  - 输入框 / 下拉框
  - 卡片 / 浮层 / 面板
- 优化应在共享层完成，尽量让页面通过继承自动受益。

### 4. 允许轻度布局节奏优化

- 可以统一基础控件高度、卡片内边距、表格行高、表头节奏等基础尺寸。
- 不改变页面整体结构，不做跨页面布局重构。
- 不重新组织复杂 Grid、区块顺序、导航结构。

## 方案边界

### 本次包含

- 升级全局设计令牌：
  - 颜色
  - 阴影
  - 圆角
  - 边框
  - 基础尺寸节奏
- 升级以下共享系统：
  - 卡片与面板
  - 表单控件
  - 按钮系统
  - 表格系统
- 通过共享样式升级影响高频页面：
  - Dashboard
  - WorkRecord
  - Issue
  - Knowledge
  - Settings

### 本次不包含

- 不重构页面布局结构。
- 不新增新的 UI 架构层。
- 不重新设计导航模式。
- 不新增动画系统或复杂动效。
- 不引入第三方 UI 库。
- 不替换现有 MVVM 结构。

## 设计方案

### 一、颜色层级精修

在 `Resources/Styles.xaml` 中保留现有靛蓝基调，但优化以下关系：

- 页面背景层与卡片表面层的对比
- hover 表面层与默认表面层的细分
- 主文本、次文本、弱文本的层级关系
- 主色、主色 hover、主色浅背景之间的明度节奏

目标：

- 视觉更克制高级，而不是更鲜艳。
- 卡片、输入框、表格、浮层在同屏时更容易形成自然层次。
- 页面不依赖粗边框来建立结构，而更多依靠表面色和阴影完成分区。

### 二、阴影与边框体系升级

精修以下共享资源：

- `ShadowSm`
- `ShadowMd`
- `ShadowLg`
- `ShadowFocus`
- `ShadowPanel`
- 各控件边框色与 hover 边框色

目标：

- 默认卡片阴影更轻、更稳。
- 悬浮卡片、浮层、侧滑面板具备更明显但克制的空间感。
- focus 状态不应显得生硬，不做强烈发光式表现。
- 边框存在感减弱，让表面和阴影承担更多结构表达。

### 三、圆角与基础尺寸节奏统一

保持现有现代化方向，但让不同类型组件拥有更清晰的圆角和尺寸规则。

重点统一：

- 按钮高度与内边距
- 输入框 / TextArea / Select 的高度与内边距
- 卡片内边距
- 表格行高、单元格留白、表头节奏
- 浮层与侧滑面板的圆角体系

目标：

- 同类控件在不同页面高度一致、节奏统一。
- 页面不会因多个控件“各自有一点点不同”而显得松散。
- 视觉和交互尺寸达到“成熟产品”的秩序感。

### 四、卡片与面板系统优化

重点升级：

- `Card`
- `CardElevated`
- `CardHover`
- `SidePanel`
- `FloatingPanel`
- `OverlayBackdrop`
- `CalendarOverlay`

优化方向：

- 卡片更像产品容器，而不是普通边框盒子。
- Hover 卡片具备明确但克制的反馈。
- 浮层、侧边抽屉与普通卡片之间形成更清楚的层级差。
- 面板的内边距与标题节奏更统一。

预期结果：

- Dashboard 和 WorkRecord 这类信息密集页面会明显更有“产品化容器感”。
- 设置页、知识库、问题页中的面板会更统一。

### 五、表单控件系统优化

重点升级：

- `Input`
- `TextArea`
- `Select`
- `Label`
- `SearchInput`

优化方向：

- 统一输入控件高度和内边距。
- 提升输入框边框、焦点态、背景层次的精致度。
- 下拉框视觉与文本输入框统一为同一系统。
- 标签文本的层级更稳定，不让表单显得过于松散或生硬。

预期结果：

- WorkRecord、Issue、Knowledge、Settings 的表单区会变得更整齐、更专业。
- 录入体验更接近成熟桌面产品，而不是默认控件皮肤。

### 六、按钮系统优化

重点升级：

- `BtnPrimary`
- `BtnSecondary`
- `BtnSuccess`
- `BtnWarning`
- `BtnDanger`
- `BtnGhost`

优化方向：

- 拉开主按钮和次按钮的层级关系。
- 让 ghost 按钮更轻，但保持明确可点击感。
- 状态按钮保持语义清晰，同时不破坏全局气质。
- 统一按钮高度、圆角、padding 和 hover/pressed 节奏。

预期结果：

- 页面主操作更聚焦。
- 次操作不会和主操作抢权重。
- 整个项目的操作反馈更细腻。

### 七、表格系统优化

重点升级：

- `ModernGrid`
- `ModernGridColumnHeader`
- `ModernGridCell`
- `ModernGridRow`
- `GridTextTrim`
- `ModernToolTip`

优化方向：

- 表头更像信息分区，而不是普通小字标题行。
- 行高、单元格左右留白、hover 反馈更统一。
- 截断文本与悬停查看的交互体验保持一致。
- 状态列、标签列、进度列的整体观感更稳定。

预期结果：

- Dashboard、WorkRecord、Issue 等页面的数据区会更耐看。
- 表格会成为产品界面的“高级信息面板”，而不是功能性控件集合。

## 页面影响预期

### Dashboard

- 统计卡、图表区、表格区之间的层级更明确。
- 首页第一屏会更有主次和空间感。

### WorkRecord

- 表单区节奏更统一。
- 表格与报告预览区的关系更协调。
- 页面在信息较多时仍保持清晰和精致。

### Issue / Knowledge

- 列表与表单风格更接近同一设计语言。
- 状态标签、字段输入、按钮系统整体更稳定。

### Settings

- 不同控件类型的气质更统一。
- 开关、按钮、选择器、分组面板更像同一个产品系统。

## 验证标准

### 修复前

- 全局样式虽然统一，但精致度与层级感仍有提升空间。
- 页面之间在卡片、表单、表格气质上存在轻微割裂。
- 高使用频率控件缺少更成熟的节奏与高级感。

### 修复后

- 不改页面结构的前提下，整体观感明显更高级。
- 颜色、阴影、圆角、尺寸节奏更稳定。
- 卡片、表单、按钮、表格四大系统形成统一设计语言。
- Dashboard、WorkRecord、Issue、Knowledge、Settings 页面之间的视觉一致性增强。

### 边界场景

- 不能因为追求高级感而降低信息密度可读性。
- 不能让 hover / focus 反馈过强，影响桌面工具的专业感。
- 不能破坏现有绑定、模板结构和组件复用关系。
